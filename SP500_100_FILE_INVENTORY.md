# SP500-100 File Inventory

This file lists which files are needed for the current 100-stock workflow in this workspace.

## Core files required for the SP500-100 workflow

These are the minimum project files needed to run the current 100-stock pipeline end-to-end.

1. `requirements.txt`
   - Python package list for the project environment.

2. `load_sp500_100.py`
   - Builds the 100-stock dataset and writes the files used by training.

3. `Staged_MARL_Training.py`
   - Current main training script for the 100-stock staged MARL workflow.
   - Reads the generated data files below.

4. `data/sp500_100_prices.csv`
   - Generated by `load_sp500_100.py`.
   - Required input for `Staged_MARL_Training.py`.

5. `data/processed/lexical_matrix_100.csv`
   - Generated by `load_sp500_100.py`.
   - Required input for `Staged_MARL_Training.py`.

6. `data/processed/sector_map_100.json`
   - Generated by `load_sp500_100.py`.
   - Required input for `Staged_MARL_Training.py`.

## Practical summary

If you only want to run the current SP500-100 training workflow, keep and use:

- `requirements.txt`
- `load_sp500_100.py`
- `Staged_MARL_Training.py`
- `data/sp500_100_prices.csv`
- `data/processed/lexical_matrix_100.csv`
- `data/processed/sector_map_100.json`

If the three generated data files already exist, you can run `Staged_MARL_Training.py` directly without rerunning `load_sp500_100.py`.
